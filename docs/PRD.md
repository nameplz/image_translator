# PRD: Image Translator

## 1. 제품 개요

Image Translator는 만화·웹툰 이미지에서 텍스트를 인식하고 번역한 뒤, 원본의 의미와 시각적 맥락을 보존한 결과 이미지를 생성하는 데스크톱 애플리케이션이다.

제품의 최우선 목표는 처리 속도가 아니라 **번역 결과와 최종 이미지의 품질**이다. 자동 처리 결과는 품질 검증을 통과해야 하며, 불확실하거나 기준을 통과하지 못한 영역은 사용자에게 명확히 표시되어야 한다.

## 2. 대상 사용자

- 한국어·영어·일본어 만화와 웹툰을 번역하려는 개인 사용자
- 캐릭터 말투와 작품 용어의 일관성을 유지하며 여러 페이지를 번역하려는 사용자
- 자동 번역 결과를 검토하고 자연어로 수정 지시를 제공하려는 사용자
- 번역, OCR, 검증, 인페인팅 provider를 교체하거나 확장하려는 개발자

## 3. 핵심 가치와 우선순위

우선순위는 다음 순서를 따른다.

1. 원문의 의미, 감정, 화자 관계 보존
2. 목표 언어에서 자연스러운 현지화
3. 작품 전체의 이름, 호칭, 말투, 용어 일관성
4. 가로쓰기·세로쓰기·우→좌 열 순서의 정확한 해석
5. 원본 배경과 텍스트 스타일을 보존한 결과 이미지
6. 사용자가 검토·수정·재검증할 수 있는 흐름
7. 처리 속도와 API 비용

## 4. 지원 범위

### MVP

- PySide6 기반 데스크톱 GUI
- 한국어·영어·일본어 상호 번역
- 단일 이미지 열기, 처리, 검토, 수정, 저장
- 가로쓰기, 일본어 세로쓰기, 우→좌 열 순서, 회전 텍스트 인식
- 대사, 내레이션, 효과음, 표지·장식 텍스트, 루비·후리가나 분류
- 주 OCR과 위험 영역 보조 OCR 교차 확인
- DeepL, Gemini, OpenAI, Grok 번역 adapter
- Gemini, OpenAI, Grok 검증 adapter
- LangGraph 기반 번역 품질 검증, 재번역, 사용자 검토 중단·재개
- LangGraph 기반 최종 이미지 품질 검증 및 재처리
- 프로젝트 단위 캐릭터·호칭·말투·용어 컨텍스트
- 자연어 수정 지시를 구조화된 변경 계획으로 변환
- 작업 진행 단계와 전체 진행률 표시
- 실행 전 output 경로 지정 및 완료 후 Save As
- 앱 재시작 후 검토 대기 작업 재개

### MVP 이후

- 폴더 및 작품 단위 batch 처리
- 다중 페이지 대화 흐름 자동 분석
- 추가 OCR, 번역, 검증, 인페인팅 provider
- 사용자 정의 렌더 테마와 고급 수동 편집 도구
- Windows PyInstaller 배포와 설치 프로그램

### 명시적 제외

- 범용 채팅 assistant
- 자유로운 자연어 도구 실행
- `@경로` 기반 자연어 명령
- 사용자 승인 없는 프로젝트 규칙 변경
- 사용자가 지정하지 않은 provider 자동 fallback
- reviewer가 최종 번역문을 직접 생성하는 구조
- 사용자 이미지의 클라우드 저장소 동기화

## 5. 핵심 사용자 시나리오

### 5.1 이미지 번역

1. 사용자가 이미지 파일을 연다.
2. 앱이 원본 이미지와 감지된 기본 언어를 표시한다.
3. 사용자가 목표 언어, OCR, 번역 provider, 검증 provider를 선택한다.
4. 사용자가 visual mode와 클라우드 이미지 전송 여부를 선택한다.
5. 사용자가 결과 이미지의 output 경로를 지정한다.
6. 사용자가 번역을 실행한다.
7. 앱은 현재 단계, 전체 진행률, 상태 메시지를 표시한다.
8. 자동 품질 검증을 통과하지 못한 영역만 검토 대기로 표시한다.
9. 모든 필수 검토가 끝나면 결과 이미지 저장이 활성화된다.

수용 기준:

- OCR, provider 호출, graph, 인페인팅, 렌더링, 저장은 GUI thread에서 실행하지 않는다.
- 진행률은 감소하지 않으며, 사용자 검토 대기 상태를 명확히 표시한다.
- 미해결 `error`/`critical` 이슈 또는 필수 사용자 확인 대기가 있으면 일반 저장을 차단한다.
- 품질 gate 통과 전에는 지정한 output 경로에 파일을 생성하지 않는다.

### 5.2 세로쓰기와 혼합 레이아웃 번역

1. 앱은 일본어 세로쓰기의 열 내부 상→하, 열 사이 우→좌 순서를 분석한다.
2. 한 페이지에 존재하는 가로쓰기, 세로쓰기, 회전 효과음을 별도로 분류한다.
3. 루비·후리가나는 본문과 연결 관계를 보존한다.
4. 번역은 올바른 reading order와 대화 그룹을 사용한다.
5. 결과는 목표 언어의 표준 쓰기 방향을 우선하여 배치한다.

수용 기준:

- 좌표 정렬만으로 reading order를 결정하지 않는다.
- reading order가 불확실하면 자동 승인하지 않는다.
- 루비는 기본적으로 번역하지 않으며 표시 여부를 사용자가 선택할 수 있다.

### 5.3 결과 검토와 자연어 수정

1. 사용자가 결과 이미지 또는 특정 영역을 선택한다.
2. 사용자가 “오른쪽 위 대사를 존댓말로 바꿔줘”와 같은 수정 지시를 입력한다.
3. 앱은 대상 영역, 적용 범위, 이전 값, 새 값, 재검증 단계를 포함한 변경 계획을 표시한다.
4. 사용자가 계획을 승인한다.
5. 앱은 수정 후 필요한 번역 또는 이미지 품질 검증을 다시 실행한다.
6. 승인된 변경은 새 revision으로 저장된다.

수용 기준:

- 자연어 지시는 허용된 편집 액션으로만 변환한다.
- 대상 또는 범위가 불명확하면 사용자 확인 없이 적용하지 않는다.
- 모든 변경은 적용 전 미리보기와 승인을 거친다.
- revision diff, Undo, Redo, 이전 결과 복원을 지원한다.

### 5.4 작업 중단과 재개

1. 사용자가 작업을 취소하거나 앱을 종료한다.
2. graph checkpoint와 작업 상태가 로컬에 안전하게 저장된다.
3. 앱 재실행 시 재개 가능한 작업을 표시한다.
4. 사용자는 마지막 안전 checkpoint부터 작업을 재개한다.

수용 기준:

- 취소는 실패가 아닌 `CANCELLED` 상태로 처리한다.
- checkpoint에는 API key, 이미지 배열, provider raw payload를 저장하지 않는다.
- 재개 시 완료된 provider 호출을 불필요하게 반복하지 않는다.

## 6. 번역 및 품질 정책

- 기본 번역 스타일은 의미·감정·캐릭터성을 보존하는 자연스러운 현지화다.
- 모든 번역 영역은 text 기반 AI 품질 검증을 받는다.
- visual mode가 활성화되면 모든 영역을 페이지 전체 이미지와 crop을 사용해 추가 검증한다.
- visual mode가 비활성화되면 이미지나 crop을 외부 provider에 전달하지 않는다.
- reviewer는 판정, 점수, 근거, 개선 지시만 반환한다.
- 영역별 자동 재번역은 최대 2회다.
- 자동 수정 후에도 기준을 통과하지 못하면 해당 영역만 사용자 검토로 전환한다.
- 번역 provider와 검증 provider는 다르게 선택하도록 권장하고, 같으면 경고한다.
- 자동 fallback은 사용자가 설정한 순서가 있을 때만 허용한다.

세부 품질 기준은 [QUALITY.md](QUALITY.md)를 따른다.

## 7. 결과 이미지 정책

- 인페인팅은 영역 난이도와 검증 결과에 따라 backend를 escalation할 수 있다.
- 렌더링은 원본 스타일을 근사하되 목표 언어 가독성과 영역 적합성을 우선한다.
- overflow, clipping, contrast, glyph, writing mode 오류는 결정론적으로 검사한다.
- visual mode가 활성화되면 인페인팅 잔상과 시각적 자연스러움을 AI가 추가 검사한다.
- 최종 이미지 자동 수정은 최대 2회다.
- 미해결 `error`/`critical` 이슈 또는 필수 사용자 확인 대기가 있으면 일반 export를 차단하고, 사용자가 경고를 확인한 경우에만 강제 export를 허용한다.

## 8. 저장 정책

- 사용자는 번역 실행 전에 output 파일 경로를 지정할 수 있다.
- 완료 후 Save As로 경로를 변경할 수 있다.
- 자동 저장은 사용자가 명시적으로 활성화한 경우에만 수행한다.
- 기본 포맷은 PNG이며 JPEG와 WebP를 지원한다.
- 입력 이미지와 같은 경로로 덮어쓰지 않는다.
- 기존 파일 덮어쓰기 전에는 사용자 확인이 필요하다.

## 9. 성공 기준

MVP는 다음 조건을 모두 만족해야 한다.

- 한국어·영어·일본어 상호 번역이 동작한다.
- 가로쓰기, `vertical_rl`, 회전 텍스트가 올바른 reading order로 처리된다.
- 모든 번역 영역이 구조 검증과 AI 품질 검증을 거친다.
- 미해결 blocking 품질 이슈가 일반 export에 섞이지 않는다.
- 자연어 수정은 구조화된 계획 승인 후 적용되고 재검증된다.
- 앱 재시작 후 사용자 검토 대기 작업을 재개할 수 있다.
- 작업 진행 단계와 전체 진행률을 항상 확인할 수 있다.
- mock provider 기반 전체 workflow integration test가 통과한다.
- `pytest --cov=src --cov-fail-under=80`, `ruff`, `mypy`가 CI gate로 동작한다.
